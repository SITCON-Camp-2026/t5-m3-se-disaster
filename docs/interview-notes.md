# 使用者訪談紀錄

> 這份文件是 Codex 依照目前對話先整理的草稿。請由人類刪除、修改或補充後，才當成正式訪談紀錄。

## 我的 Phase 0 prototype 現況

- 目前工作台能顯示 Phase 0 原始資訊、資訊取得方式、查核狀態與需要人工確認的地方。
- 畫面中的分類與派工準備度仍是整理草稿，不代表資訊已確認，也不代表可以直接派工。
- 使用者希望 v1 更像「資訊整理者」使用的服務，幫忙判斷資料能不能被信任、還缺什麼資訊。
- 這次訪談前特別關注：如果使用者只是轉發網路貼文，資訊整理者要怎麼檢查來源是否正確。

## Persona 1：回報者

### 我想完成的事

- 快速把看到的資訊送出，不想被太多欄位卡住。
- 如果是轉發貼文，希望能貼上原始連結、截圖或補充說明。

### 我看不懂的地方

- 不一定知道「原始來源」和「轉發者」要分開填。
- 可能不清楚什麼資訊算第一手，什麼資訊只是聽說或轉述。

### 我最怕的錯誤

- 因為填寫太慢，導致重要線索沒有被送出。
- 被系統要求判斷真偽，但自己其實沒有能力查證。

### 我不敢直接相信的資訊

- 只有截圖、沒有原文連結的貼文。
- 轉發者說「朋友看到」或「群組有人說」但沒有來源的內容。

### 我希望系統提醒我的事

- 系統應提醒我保留原始貼文網址、發文時間、發文者帳號，以及我是從哪裡看到這則資訊。
- 系統應提醒我不要把轉發內容改寫成自己親眼看到的事。

### 我想反問開發者的一個問題

- 如果我只能提供截圖，沒有原始連結，系統要怎麼標示這筆資料？

## Persona 2：資訊整理者

### 我想完成的事

- 把原始資訊整理成「可以繼續確認的線索」，並清楚標出哪些地方不能直接相信。
- 對轉發網路貼文建立檢查清單：原始貼文、發文者、發文時間、轉發者、是否有第二個獨立來源。
- 區分「資訊取得方式」與「查核狀態」，避免把社群貼文、官方截圖或志工轉述直接當成已確認。

### 我看不懂的地方

- 目前如果只看到 sourceType，仍不一定知道它是第一手、第二手，還是多次轉傳。
- 若貼文內容提到「官方說」但沒有連結或日期，無法判斷是不是舊訊息或斷章取義。

### 我最怕的錯誤

- 把轉述、截圖或社群貼文顯示成已確認事實。
- 把缺少來源、時間、地點的資訊直接交給行動者，造成錯誤派工。
- AI 因為文字看起來合理，就自動補出原文沒有的來源可信度。

### 我不敢直接相信的資訊

- 沒有原始連結的轉發貼文。
- 只有「有人說」「群組傳」這類描述，沒有可追溯來源的資訊。
- 沒有發文時間或事件時間的貼文，因為可能是過期資訊。
- 轉發者不是當事人，也沒有說明與當事人的關係。

### 我希望系統提醒我的事

- 先把轉發貼文標成「未確認」或「來源待確認」。
- 保留原始內容，不要在整理時改寫成確定語氣。
- 提醒我追問：原始貼文在哪裡、發文者是誰、發文時間是何時、是否有現場證據、是否有第二個獨立來源。
- 顯示「目前不能直接變成任務」的原因，而不是只顯示分類結果。

### 我想反問開發者的一個問題

- v1 要不要把「原始來源」「轉發來源」「查核狀態」「人工確認紀錄」拆成不同欄位？

## Persona 3：行動者

### 我想完成的事

- 只接收已經被整理者確認過、可以採取下一步的資訊。
- 在行動前知道這筆資訊還缺哪些關鍵資料，例如地點、時間、需求是否仍存在。

### 我看不懂的地方

- 如果畫面只顯示分類，可能看不出這是 AI 草稿還是人工確認後的結果。
- 如果來源寫「網路貼文」，不一定知道可信度高低。

### 我最怕的錯誤

- 根據未確認貼文出發，結果到現場才發現資訊是舊的、錯的或不完整。
- 把轉發者當成當事人，漏掉確認同意與現場狀況。

### 我不敢直接相信的資訊

- 沒有明確時間、地點、需求對象的資訊。
- 只靠轉發截圖或留言整理出的任務。

### 我希望系統提醒我的事

- 這筆資訊是否已經由人確認。
- 哪些欄位仍缺漏，因此不能直接行動。
- 如果只是轉發貼文，應明確標示「不可直接派工」。

### 我想反問開發者的一個問題

- 我看到資訊時，怎麼知道它只是整理草稿，還是已經可以採取行動？

## 人類補充

- 我目前想以「資訊整理者」作為 v1 主要使用者。
- 我特別想補更多關於「轉發網路貼文如何確認來源正確性」的判斷。
- 人類仍需要確認：哪些來源檢查項目是 v1 必做，哪些只是訪談後的備忘。
